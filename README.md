# Prime Cars

A multilingual car-import and dealership web platform for the Armenian market ([primecars.am](https://primecars.am)). Visitors can browse available cars, estimate import/customs costs with a calculator, apply for partnership, and view partners. An authenticated admin panel manages cars, users, calculator settings, and notifications.

Built with **Next.js 14 (App Router)**, **TypeScript**, **Tailwind CSS**, and **next-intl**.

## Features


- **Public site** — homepage, car catalog (`/cars`, `/cars/[id]`), import-cost calculator, partners, and an application/registration form.
- **Internationalization** — three locales (`hy` Armenian (default), `en`, `ru`) via `next-intl` with `localePrefix: "always"`. Every route is prefixed (e.g. `/hy/cars`).
- **Admin panel** (`/[locale]/admin`) — manage cars, available cars, users, calculator settings, and notifications. Protected by token-based auth and role-based access control (RBAC).
- **Import calculator** — estimates customs/import costs for vehicles.
- **Email notifications** — registration/partnership requests are emailed to the admin via [Resend](https://resend.com).
- **SEO** — sitemap and `robots.txt` generated by `next-sitemap`; OG images served from `/public/og`; AI crawlers disallowed.
- **Security headers** — HSTS, `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, and a restrictive `Permissions-Policy` set in `next.config.js`.

## Getting started

### Prerequisites

- Node.js 18+ (Node 20 recommended)
- npm

### Install & run

```bash
npm install
npm run dev
```

The app runs at [http://localhost:3000](http://localhost:3000) and redirects to the default locale (`/hy`).

### Available scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Start the development server |
| `npm run build` | Production build (runs `next-sitemap` afterwards via `postbuild`) |
| `npm run start` | Start the production server |
| `npm run lint` | Run ESLint |
| `npm run lint:fix` | Run ESLint with `--fix` |
| `npm run type-check` | Type-check with `tsc --noEmit` |
| `npm run zip` | Build and package the app into `build.zip` (excludes `node_modules`/`.git`) |

## Environment variables

Create a `.env` (or `.env.local`) file. The backend API base URL is currently hardcoded in `src/i18n/config.ts` (`API_BASE_URL`).

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_MAINTENANCE_MODE` | No | Toggle maintenance mode (`true`/`false`). |
| `NEXT_PUBLIC_SITE_URL` | No | Canonical site URL for sitemap generation. Defaults to `https://primecars.am`. |
| `RESEND_API_KEY` | Yes (for email) | API key for sending registration emails via Resend. |
| `RESEND_FROM_EMAIL` | No | Sender address. Defaults to `onboarding@resend.dev`. |
| `REGISTRATION_NOTIFICATION_EMAIL` | No | Recipient for registration/partnership notifications. |

## Internationalization

- Locales: `hy` (default), `en`, `ru` — defined in `src/i18n/config.ts`.
- Translation messages live in `src/messages/{locale}.json`.
- Routing uses `localePrefix: "always"`, so `/hy/*` is the canonical Armenian path. Do **not** add locale redirect rules in `next.config.js` — they collide with the next-intl middleware.

## Backend

The frontend talks to an external backend API (`API_BASE_URL` in `src/i18n/config.ts`). Public, unauthenticated reads (car listings, detail pages) use the `publicFetchJson` helper in `src/lib/cars/`. Authenticated admin requests use token-based auth (`src/lib/auth/token.ts`) with access/refresh tokens stored in `localStorage`.

## Deployment

Standard Next.js build. Remote image hosts (Unsplash, Railway/Google Cloud Storage) are allowlisted in `next.config.js`. Run `npm run build` to produce a production build with a generated sitemap and `robots.txt`.
